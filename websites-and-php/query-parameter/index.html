?i=1 Parameter In My URL

When you visit a website hosted by us, you may see a parameter appended to the URL. It can be ?i=1 or ?i=2 or nonexistent but still maybe present in other forms.

What does this parameter do?​

This parameter helps ensure that your website is only accessible by Web Browsers.

When you access your website, you may notice that the ?i=1 parameter is appended. This means it is the first attempt to verify your browser.

This is achieved by setting a cookie (called _test) to observe if the JavaScript challenge is completed and passed. If the parameter shows ?i=2, this means that the JavaScript security system cannot validate the cookie.

This can be because the cookie has not been found or it did not match the value provided by the JavaScript system.

If the parameter reaches 3+ and the browser has not been validated, you will be redirected to a Google Support article with instructions regarding enabling cookies in your browser. This makes sure the browser does not get stuck in a redirect loop.

Can I remove this parameter?​

Because we (PleasantFree) provide our service free of charge, we must ensure that our service isn’t abused. Our service is for Website hosting only, therefore we do not permit programmatic access.

This security system is mandatory and cannot be turned off, blocked, or disabled, even with the .htaccess code.

If you need this system to be removed, please upgrade to premium hosting where it is not present.
